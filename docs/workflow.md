# AI Coding Task Workflow

本文件說明 **AI Coding Relay** 的使用方式，協助新使用者快速上手。

---

## 1. 工具用途

AI Coding Relay 是一個**單機任務管理工具**，用來管理「交給 AI（Claude Code）執行的程式開發任務」。

它解決的問題是：當你把開發任務交給 AI 時，需要反覆「整理需求 → 貼給 AI → 收回回覆 → 審查 → 要求修正」。這個過程的脈絡很容易散落在各處。本工具把每個任務的需求、限制、AI 回覆、驗收結果集中保存，並一鍵產生要貼給 AI 的 Prompt。

- 所有資料儲存在**瀏覽器的 localStorage**（key：`ai-coding-relay:task-store`），不會上傳雲端。
- 因為資料只在本機，請善用「匯出 / 匯入」做備份（見第 5 節）。
- 介面分為兩區：左側 **任務側欄**（清單、搜尋、篩選、排序、統計、匯出入），右側 **任務詳情**（編輯欄位、產生 Prompt、操作任務）。

---

## 2. 任務生命週期

一個任務從建立到完成，典型流程如下：

```
建立任務
   ↓
複製 Claude Prompt  ──→  貼到 Claude Code 執行
   ↓
貼回 Claude 回覆（填入「Claude 回覆」欄位）
   ↓
複製審查 Prompt  ──→  貼給審查者（人或另一個 AI）做 code review
   ↓
設定驗收結果（reviewResult）
   ↓
┌─ 通過 ──────────────→ 狀態改為「已完成」→（可選）封存
│
└─ 需修改 ─→ 複製修正 Prompt ──→ 貼回 Claude 繼續修正
                ↑                        │
                └────────────────────────┘（重複直到通過）
```

逐步說明：

1. **建立任務** — 側欄點「＋ 新增任務」，填寫標題、類型、原始需求等。可選用內建模板快速帶入需求骨架。
2. **複製 Claude Prompt** — 在任務詳情點此按鈕，把整理好的需求 Prompt 貼到 Claude Code。
3. **貼回 Claude 回覆** — Claude 完成後，把它的回覆貼進詳情的「Claude 回覆」欄位（`claudeResponse`）。
4. **複製審查 Prompt** — 產生一份請審查者檢查「是否符合需求、是否超出範圍」的 Prompt，交給人或另一個 AI 審查。
5. **設定驗收結果** — 依審查結論，把 `reviewResult` 設為「通過」或「需修改」。
6. **若需修改** — 設為「需修改」後，用「複製修正 Prompt」產生帶有驗收結果與下一步的 Prompt，貼回 Claude 繼續修正。修正完成後回到步驟 3，直到驗收通過。
7. **完成或封存** — 驗收通過後，把狀態改為「已完成」。長期不需再看的任務可「封存」收進「已封存」分頁，保持「進行中」清單乾淨。

## 3. 欄位說明

| 欄位 | 說明 |
|------|------|
| **status**（狀態） | 任務進度：`待處理` / `進行中` / `已完成`。在詳情列以下拉選單切換，影響側欄狀態篩選與統計。 |
| **priority**（優先級） | 重要程度：`⬆ 高` / `➡ 中` / `⬇ 低`。預設為「中」。可作為排序依據。 |
| **dueDate**（截止日） | 任務截止日期。設定後，側欄卡片會顯示日期徽章，逾期會標紅、今天到期會標黃。 |
| **tags**（標籤） | 自由分類標籤，以逗號分隔輸入（例：`frontend, bug, urgent`）。可用標籤篩選任務。 |
| **project**（專案分類） | 任務所屬的專案名稱（例：`my-app`），用來分組。留空視為「未分類」。 |
| **projectPath**（專案路徑） | 專案在本機的實際資料夾路徑（例：`/Users/ryan/projects/my-app`）。會一併寫進 Claude Prompt，方便 AI 知道在哪個目錄工作。 |
| **claudeResponse**（Claude 回覆） | 貼上 Claude 對這個任務的回覆內容。會被帶入「審查 Prompt」與「修正 Prompt」。 |
| **reviewResult**（驗收結果） | 審查結論：`未驗收` / `通過` / `需修改`。是決定要「完成」還是「複製修正 Prompt」的依據。 |
| **nextActions**（下一步） | 記錄接下來要做的事，可多行。會被帶入「修正 Prompt」，讓 Claude 知道下一輪重點。 |
| **summary**（任務摘要） | 任務完成後的回顧筆記：做了什麼、改了哪些檔案、遇到什麼問題、怎麼解、如何驗收。供日後查閱。 |

> 此外建立任務時還會填寫：**原始需求**、**目標檔案**、**禁止修改範圍**、**限制條件**、**驗收條件**——這些是組成 Prompt 的主要內容。

## 4. 三種 Prompt 按鈕用途

任務詳情標題列的「Prompt 工具」分組有三個按鈕，點擊後內容會複製到剪貼簿（按鈕短暫顯示「✓ 已複製」）。空白欄位會自動略過。

### 複製 Claude Prompt
產生**初次交辦給 Claude** 的 Prompt。內容包含：任務標題、專案分類、專案路徑、Tags、原始需求、允許修改檔案、禁止修改範圍、限制條件、驗收條件。
→ 用於任務生命週期的「開工」階段。

### 複製審查 Prompt
產生**請審查者 code review** 的 Prompt。以「請幫我審查這次 Claude Code 的修改是否符合需求」開頭，包含：原始需求、允許修改檔案、禁止修改範圍、限制條件、Claude 回覆，並固定附上一段檢查清單（是否符合需求、是否超出範圍、是否違反限制、是否有型別/安全/維護性問題、是否需補測試）。
→ 用於收到 Claude 回覆後、決定驗收結果之前。

### 複製修正 Prompt
產生**請 Claude 繼續修正** 的 Prompt。以「請根據以下驗收結果，繼續修正這個任務」開頭，包含：任務標題、專案資訊、原始需求、Claude 回覆、驗收結果、下一步、限制條件、驗收條件。
→ 用於 `reviewResult` 設為「需修改」後，把問題回饋給 Claude 進行下一輪。

## 5. 匯出 / 匯入備份

因為所有資料只存在瀏覽器 localStorage，**換瀏覽器、清快取或換裝置都會遺失**，請定期備份。側欄上方有兩個按鈕：

- **↑ 匯出** — 把目前所有任務與回合資料下載成一個 JSON 檔（檔名格式 `tasks-backup-YYYY-MM-DD.json`）。
- **↓ 匯入** — 選擇先前匯出的 JSON 檔還原資料。

> ⚠️ **匯入會覆蓋目前所有資料**，動作前會跳出確認視窗。建議定期匯出保存；要在新裝置使用時，先在舊裝置匯出、再到新裝置匯入。

## 6. 篩選、搜尋、排序

側欄提供多層工具，協助在大量任務中快速定位。它們**可疊加使用**（搜尋 + 各種篩選同時生效）。

### 分頁
- **進行中 / 已封存** — 切換檢視未封存或已封存的任務。下方所有篩選、統計都只針對目前分頁。

### 搜尋
- 上方搜尋框會比對任務標題、需求、回覆等內容；輸入後清單即時過濾，並顯示「共 N 筆」。找不到時顯示「找不到符合的任務」。

### 篩選（共五組，可同時套用）
- **狀態**：全部 / 待處理 / 進行中 / 已完成。
- **截止日**：全部 / 逾期 / 今天到期 / 有截止日 / 未設定。
- **標籤**：列出目前出現過的所有標籤，點選只看含該標籤的任務（任務有標籤時才顯示這一列）。
- **專案**：全部 / 未分類 / 各專案名稱。
- **驗收**：全部 / 未驗收 / 通過 / 需修改。

> 若某個標籤或專案因改名、清空而消失，對應篩選會自動視為「全部」，清單不會卡在空白。

### 排序
「排序」下拉選單可選：
- **優先級** — 高 → 中 → 低。
- **截止日** — 日期早的在前，未設定截止日的排最後。
- **狀態** — 待處理 → 進行中 → 已完成。
- **建立時間** — 最新建立的在前。

### 統計
篩選列下方有一排統計徽章，反映**目前分頁**的數量：全部、待處理、進行中、已完成、逾期、今天、⬆高優、未驗收、通過、需修改。可快速掌握整體狀態。

## 7. AI Workflow（CE Pipeline）— 進階自動化入口

任務詳情中的 **AI Workflow** 區塊，是除了第 2 節手動 Relay 之外、較自動化的 CE（Compound Engineering）流程入口。當任務有設定 **Project Path**（且本機有跑 `pnpm runner:local`）時可使用。

- **Pipeline mode 是主要入口**：按 **「Run CE Pipeline」** 會一鍵串接 Readonly → Work → Review → Commit → Compound → Save，只在 **Work 前** 與 **Commit 前** 兩個高風險點停下等你確認；不會自動 push。詳細操作見 `docs/ce-workflow-smoke-test.md` 第 5 節。
- **Advanced manual controls 是 fallback / debug**：預設收合，內含舊的逐步手動流程（CE Readonly / Work / Review / Fix / Commit checkpoint）。僅在 Pipeline 失敗、Review needs_fix 需手動 Fix、或想單獨重跑某階段時展開使用，不是日常主流程。
- **Workflow details 是查看 / 編輯細節**：預設收合，展開後可檢視與編輯 Brainstorm / Plan / Audit / Work · Review / Compound 各階段欄位與 Copy Prompt。
- **Summary panel 是主要狀態摘要**：右側面板顯示進度、下一步、Project Path、Audit checklist、Review 結果、commit hash、changed files 等精簡資訊。
- **completed workflow 會 disable「Run CE Pipeline」**：已完成（已 commit + Review passed + Compound 已記錄）的任務不能重跑，避免再跑到 `nothing_to_commit`；要重跑請建立新任務。
